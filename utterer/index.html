Utterer


The Utterer package comes with three main components: Utterer Dialogue System, Utterer Languages, and Utterer Dynamics.



Utterer Dialogue System (UDS)



Utterer Dialogue System is built to format itself based on your input so you can easily create and control single person/multiple person dialogue/subtitles/captions. UDS is fully customizable. Add your own fonts and textures or use the ones that come with the package.



The Utterances array contains the utterances for your dialogue/subtitles, plus content to display for each utterance.



Utterance Content


-Icon of the speaker.
-The speaker text.
-The dialogue/subtitle text.
-Sound played during this dialogue/subtitle.
-Duration the dialogue/subtitle is shown.
-Delay til the next dialogue/subtitle is shown.



UDS Settings



-Delay UDS from starting the dialogue/subtitle sequence.
-Fade the system in and out at the beginning and end of the dialogue/subtitle sequence.
-Fade in and out all content between each utterance.
-Reformat UDS into a subtitle system rather than a dialogue system.
-Set to interactive so UDS progresses dialogue through user input.
-Set to typing animation so UDS adds one letter at a time like a typewriter.
-Customize the size and position of Utterer.



UDS Style


-Hide the icon, next button, background, outline, or speaker.
-Choose the color and texture (sprite/image) of each in their respective fields.
-Choose the color, font, font style, font size, etc of the speaker and dialogue/subtitle texts.





Utterer Languages





Utterer Languages localization framework for Text/TextMesh Pro, AudioSource, and Image components as well as the Utterer Dialogue System. Text/UDS translations can be imported and exported as CSV data.

For general use :
Add an UttererLanguages script to any object with a Text, TextMesh Pro, Audio Source, or Image component. Text areas/audio/sprite fields will be added for each language in the list and the Text, TextMesh Pro, Audio Source, or Image component will display the text/audio/sprite of whichever language is the current language in use.

For UDS :
Add an UttererLanguages script to a UDS prefab in the scene. Speaker text areas and utterance text areas will appear in each Utterance for every language in the Languages list, along with an option for localized sound in each Utterance.




Utterer Dynamics


Utterer Dynamics contains several built in Dynamics that are formatted based on the current culture settings :

Several DateTime calls (<LongDate>, <ShortDate>, <LongTime>, etc.)
Currency based on culture settings (<c123.4567> = $123.45)
Decimals/doubles/floats, etc. based on culture settings (<1234.56789> = 1234.568)

Dynamics List
With the Dynamics List you can call a specific key and replace it with its ID string. These call and ID strings are saved into the device's persistent data in a file called DynamicsList.json.
